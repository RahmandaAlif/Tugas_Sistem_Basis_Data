Tahapan Merancang ERD

  1. Menentukan entitas
  2. Menentukan Atribut yang termasuk atribut kunci
  3. Identifikasi relasi
  4. Menentukan kardinalitas

Studi kasus : ERD Tentang Penjualan Barang Online

ERD Penjual merupakan aktifitas menjual produk, bisa berupa barang ataupun jasa. Aktifitas ini di lakukan oleh dua pihak yakni penjual dan pembeli. Pada studi kasus kali ini kita akan coba membuat perancangan basis data menggunakan pemodelan

Aturan

  1. Seorang penjual dapat menjual banyak barang
  2. Seorang pembeli dapat membeli lebih dari satu barang yang dijual penjual
  3. Stok barang akan berkurang sesuai jumlah barang yang dibeli
  4. Penjual, pembeli, dan barang dapat diidentifikasi dengan id yang berbeda(unik)

Tahapan 1 : Menentukan Entitas

  • penjual, barang, pembeli

Tahapan 2 : Menentukan Atribut

    Penjual

  • Id_Penjual : integer(pk)
  • Nama_Penjual : string
  • Email : string
  • No_telp : string

    Barang

  • Id_Pembeli : integer(pk)
  • Nama_Barang : string
  • Harga : integer
  • Stok : integer

    Pembeli

  • Id_Pembeli : integer(pk)
  • Nama_Pembeli : string
  • Alamat : integer
  • Email : integer

Tahapan 3 : Menentukan Relasi dan Kardinalitas

  • Kardinalitas relasi antara penjual dan barang adalah one to many karena seorang penjual dapat menjual banyak barang atau banyak barang dapat dijual oleh seorang penjual
  • Kardinalitas relasi antara pembeli dan barang adalah one to many karena seorang pembeli dapat membeli banyak barang atau dengan kata lain banyak barang boleh dibeli oleh seorang pembeli
